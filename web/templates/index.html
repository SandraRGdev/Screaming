Patrones de exclusión de incidencias

Las URLs que coincidan con estos patrones se excluirán de la detección de incidencias. Suelen ser áreas de administración, archivos de desarrollo e internals de frameworks que no deberían indexarse nunca.

Usa * como comodín. Ejemplos: /admin/*, *.json, /test/*
Recupera los patrones de exclusión predeterminados
ℹ️ Las exclusiones predeterminadas incluyen:
• Áreas de administración de WordPress (/wp-admin/*, /wp-content/plugins/*)
• Paneles de administración comunes (/admin/*, /administrator/*, /cpanel/*)
• Archivos de desarrollo (/.git/*, /node_modules/*, /vendor/*)
• Directorios de build y caché (/build/*, /cache/*, /tmp/*)
• Archivos de configuración (*.json, *.yaml, *.xml)
• Archivos de copia de seguridad (*.bak, *.backup, *.old)

Estilos CSS personalizados

Escribe CSS personalizado para adaptar el aspecto de ScreamingWeb. Los cambios se aplican al guardar.

Introduce reglas CSS válidas para personalizar la apariencia
💡 Consejos de CSS:
• Apunta a elementos concretos: .header, .data-table, .sidebar
• Cambia colores: background-color, color, border-color
• Modifica espaciado: padding, margin, gap
• Ajusta tipografías: font-family, font-size, font-weight
• Usa !important para sobrescribir estilos si hace falta
• Los cambios solo afectan a la interfaz
🎨 Ejemplos de personalización:

/* Cabecera morada oscura */
.header { background: linear-gradient(135deg, #667eea 0%, #764ba2 100%); }

/* Botones con acento verde */
.btn-primary { background: #10b981; }

/* Texto de tabla más grande */
.data-table { font-size: 14px; }
